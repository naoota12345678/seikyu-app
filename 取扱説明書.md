## 1. はじめに

本アプリは、見積書・納品書・請求書の作成から送信、入金管理までを一元管理する請求管理システムです。

### 主な特徴

- 見積書 → 納品書 → 請求書のワンクリック変換
- 締日請求・即時請求・定期請求に対応
- 税率別（10%/8%/軽減税率）の自動計算
- PDF生成・メール送信・送信予約
- 楽天・Amazon売上の自動連携
- Stripe決済による再請求
- 承認フロー（管理者承認）
- CSV入出力対応

### 権限について

| 権限 | できること |
|------|-----------|
| **管理者（admin）** | 全機能の利用、ユーザー管理、データ削除、承認・却下 |
| **スタッフ（staff）** | 書類の作成・編集・送信（削除・承認は不可） |

## 2. ログイン

1. アプリにアクセスすると、ログイン画面が表示されます
2. 管理者から通知された **メールアドレス** と **パスワード** を入力します
3. 「ログイン」ボタンをクリックします

> ログインに失敗した場合、エラーメッセージが表示されます。パスワードを何度も間違えるとアカウントが一時ロックされます。

## 5. 見積書一覧

見積書の作成・管理・納品書への変換を行います。

### 見積書の作成

1. 「+ 新規作成」ボタンをクリック
2. 以下の項目を入力します

| 項目 | 説明 |
|------|------|
| 取引先 | ドロップダウンから選択 |
| 事業部 | ドロップダウンから選択（任意） |
| 見積日 | 日付を入力 |
| 有効期限 | 日付を入力 |
| 明細行 | 商品を選択し、数量・単価を入力 |
| 備考 | 自由記述（任意） |

3. 明細行は「+ 行追加」で追加、各行の「×」で削除できます
4. 商品を選択すると、単価と税率が自動セットされます（取引先別単価がある場合はそちらを優先）
5. 小計・消費税・合計は税率別に自動計算されます
6. 「保存」で見積書を作成します

### ステータス

| ステータス | 意味 |
|-----------|------|
| 作成済 | 作成直後の状態 |
| 受注済 | 納品書に変換済み |
| 期限切れ | 有効期限を過ぎた見積書 |

### 操作ボタン

| ボタン | 説明 |
|--------|------|
| 編集 | 見積書の内容を修正 |
| 🖨 印刷 | 印刷プレビューを表示 |
| 📥 PDF | PDFを生成してFirebase Storageに保存 |
| 納品書作成 | 見積書の内容をもとに納品書を自動作成（ワンクリック変換） |
| 削除 | 見積書を削除（管理者のみ） |

### 検索

- 画面上部の検索バーで、取引先名または見積番号で絞り込みできます

## 6. 納品書一覧

納品書の作成・管理・請求書発行を行います。

### 納品書の作成

1. 「+ 新規作成」ボタンをクリック
2. 取引先・事業部・納品日・明細行を入力
3. 「保存」で納品書を作成

> 見積書一覧から「納品書作成」で変換した場合は、見積書の内容がコピーされます。

### ステータス

| ステータス | 意味 |
|-----------|------|
| 未請求 | 請求書が未発行 |
| 承認待ち | 承認フロー待ち |
| 請求済 | 請求書を発行済み |

### 請求書発行（即時請求の場合）

1. 「請求書発行」ボタンをクリック
2. 確認モーダルが表示されます
3. 送信予定日を設定できます（任意）
4. 「発行」をクリックすると、請求書が自動作成されます

> 締日請求の取引先の場合、「請求書発行」ボタンは表示されません。月締め管理から一括発行してください。

### 操作ボタン

| ボタン | 説明 |
|--------|------|
| 編集 | 納品書の内容を修正（未請求の場合のみ） |
| 🖨 印刷 | 印刷モード選択（納品書単体 / 納品書+請求書セット） |
| 請求書発行 | 請求書を作成（即時請求の取引先のみ） |
| 削除 | 納品書を削除（管理者のみ） |

## 7. 請求書一覧

請求書の管理・送信・入金記録を行います。

### 画面表示

- 上部に **未収残高合計** が表示されます
- 検索バーで取引先名・請求書番号で絞り込みできます

### 一覧テーブル

| 列 | 説明 |
|----|------|
| 請求書番号 | 自動採番（例: INV-202603-001） |
| 日付 | 請求日 |
| 取引先 | 取引先名 |
| 請求金額 | 税込合計 |
| 未収残高 | その取引先の現在の残高 |
| 支払期限 | 支払期日 |
| ステータス | 入金済 / 未収 / 期限超過（クリックで入金済⇔未収を切替） |
| 送信状態 | 送信済 / 予約 / 未送信 |

### 操作ボタン

| ボタン | 説明 |
|--------|------|
| 🖨 印刷 | 印刷プレビューを表示 |
| 送信 / 再送 | メールで請求書を送信（PDFリンク付き） |
| 再請求 ▼ | 未入金時の再請求（メール督促 or Stripe決済） |
| 入金記録 | 入金があった場合に記録 |
| 削除 | 請求書を削除（管理者のみ） |

### メール送信

1. 「送信」ボタンをクリック
2. 取引先のメールアドレスが自動入力されます
3. 「送信」を実行すると、PDFダウンロードリンク付きのメールが送信されます
4. 送信履歴に記録されます

### 再請求（未入金の場合）

**メール再請求：**
1. 「再請求 ▼」→「✉ メールで再請求」を選択
2. 督促メッセージのテンプレートが表示されます（編集可能）
3. 承認設定が有効な場合は「承認待ち」に登録されます

**Stripe再請求：**
1. 「再請求 ▼」→「💳 Stripeで請求」を選択
2. 手数料（3.6%）が加算された金額が表示されます
3. 確認後、Stripeの請求書と決済リンクが作成されます

### 入金記録

1. 「入金記録」ボタンをクリック
2. 入金日と入金額を入力
3. 「記録」をクリックすると、残高が自動更新されます

## 8. 月締め管理

締日請求の取引先に対して、期間内の納品書をまとめて請求書を発行する画面です。

### 使い方

1. 画面上部で **対象月** を選択します（例: 2026-03）
2. 締日請求の取引先がカード形式で一覧表示されます
3. 各カードをクリックして展開すると、その期間の納品書が表示されます

### 表示内容（取引先カード）

- 取引先名と締日（例: 月末締め、15日締め）
- 未収残高（残高がある場合）
- 期間内の納品件数と未請求件数

### 締日請求書の発行

1. カードを展開し、「請求書発行」ボタンをクリック
2. 対象期間・取引先・納品件数・合計金額を確認
3. 「発行」をクリックすると：
   - 対象期間の納品書がすべて「請求済」に更新されます
   - 複数の納品書をまとめた1枚の請求書が作成されます
   - 取引先の残高が更新されます

### 締日の仕組み

- 取引先ごとに最大4つの締日を設定できます（例: 15日・末日）
- 締日が複数ある場合、各期間ごとに請求書を分けて発行します
- 例: 15日・末日の場合
  - 1日〜15日分 → 15日締め請求書
  - 16日〜末日分 → 末日締め請求書

## 9. 売上管理

売上の集計・分析を行います。自社請求分に加え、楽天・Amazonの売上も統合表示できます。

### 表示切替

| ボタン | 説明 |
|--------|------|
| 日別 | 選択月の日別売上 |
| 月別 | 直近12ヶ月の月別売上（棒グラフ付き） |
| 年別 | 年単位の売上集計 |
| 事業部別 | 事業部ごとの12ヶ月推移 |
| 取引先別 | 取引先ごとの12ヶ月推移 |

### サマリーカード

- 当月売上合計
- チャネル別内訳（請求書 / 楽天 / Amazon）
- 取引件数
- 前月比成長率（%）

### EC売上連携

| ボタン | 説明 |
|--------|------|
| 楽天 今日同期 | 楽天RMSから当日の売上を取得 |
| Amazon 今日同期 | Amazon SP-APIから当日の売上を取得 |
| CSV 売上取込 | 外部売上データをCSVで取り込み |

**楽天同期の仕組み：**
- 楽天RMS APIで注文データを取得
- 商品金額 − クーポン値引き = 売上金額（店舗カルテと一致）
- 送料は含みません

**Amazon同期の仕組み：**
- Amazon SP-API（OrderMetrics）で売上データを取得
- 日本マーケットプレイス（A1VC38T7YXB528）が対象

**CSV取込のフォーマット：**
```
日付,売上,件数,ソース
2026-03-01,50000,3,楽天
2026-03-01,30000,2,Amazon
```
- 「日付」「売上/金額」列は必須
- 「件数」「ソース/source/チャネル」列は任意

## 10. 残高管理

取引先ごとの未収残高の確認、入金記録、取引先元帳の表示を行います。

### 画面上部

- **未入金アラート** — 支払期限を1ヶ月以上超過した取引先の一覧
- **サマリーカード**
  - 未収残高合計
  - 期限超過件数
  - 対象取引先数
  - 残高あり取引先数

### フィルター

| ボタン | 説明 |
|--------|------|
| すべて | 全取引先を表示 |
| 未入金アラートのみ | 期限超過の取引先のみ |
| 残高ありのみ | 残高がゼロでない取引先のみ |

### 取引先カード

各取引先をクリックすると展開され、以下の情報が表示されます：

- 現在の残高（赤: 未収あり、緑: 完済）
- 未払い請求書の件数
- 「要確認」バッジ（期限超過がある場合）

### 操作ボタン

| ボタン | 説明 |
|--------|------|
| 入金記録 | 入金日・入金額を入力して記録 |
| 期首残高 | 基準日と期首残高を設定（導入時の繰越残高） |
| 再請求 ▼ | メール督促 or Stripe決済で再請求 |
| 残高再計算 | 残高を再計算（期首残高 + 請求合計 − 入金合計） |

### 取引先元帳

カードを展開すると、直近20件の取引履歴が表示されます。

| 列 | 説明 |
|----|------|
| 日付 | 取引日 |
| 摘要 | 請求書番号 or 入金内容 |
| 借方 | 請求額（赤字） |
| 貸方 | 入金額（緑字） |
| 残高 | その時点の累計残高 |

- 入金行には「取消」ボタンがあり、誤入金を取り消せます（管理者のみ）

### 期首残高の設定

アプリ導入時に、それ以前の未収残高を登録する機能です。

1. 「期首残高」ボタンをクリック
2. 基準日と金額を入力
3. 「保存」をクリック

> 期首残高を設定すると、元帳の最初の行に表示され、以降の残高計算の起点となります。

## 11. 承認待ち

承認フローが有効な場合、請求書発行・再請求などが「承認待ち」に登録されます。

### 承認が必要なケース（設定による）

- 請求書の手動発行
- Cron（自動処理）による定期請求の発行
- 再請求（メール・Stripe）

### 画面構成

**未承認タブ：**
- 承認待ちの項目が一覧表示されます
- 各行にタイプバッジが表示されます

| バッジ | 意味 |
|--------|------|
| 請求書発行 | 通常の請求書発行 |
| 定期 | 定期請求による発行 |
| 締日 | 締日請求による発行 |
| ✉再請求 | メール再請求 |
| 💳Stripe | Stripe再請求 |

**操作（管理者のみ）：**
- 「承認・発行」— 請求書を発行し、設定に応じてメールを自動送信
- 「却下」— 承認を拒否

**処理済みタブ：**
- 承認・却下済みの項目が表示されます（直近20件）

## 12. 定期請求

毎月・隔月・四半期で自動的に請求書を発行する設定を行います。

### 定期請求の登録

1. 「+ 新規作成」ボタンをクリック
2. 以下の項目を入力します

| 項目 | 説明 |
|------|------|
| 取引先 | ドロップダウンから選択 |
| 事業部 | ドロップダウンから選択（任意） |
| 品名 | 請求する商品・サービス名 |
| 数量 | 数量 |
| 単位 | 単位（例: 式、個） |
| 単価 | 単価 |
| 税率 | 10% / 8% / 0% / カスタム |
| サイクル | 毎月 / 隔月 / 四半期 |
| 請求日 | 末日 or 1日〜28日 |
| 送信モード | 自動送信 / 確認後送信 / 手動 |
| 開始月 | 請求を開始する月 |
| 終了月 | 請求を終了する月（空欄 = 無期限） |

### 一覧表示

| 列 | 説明 |
|----|------|
| 取引先 | 取引先名 |
| 品名 | 請求内容 |
| 金額 | 税込金額 |
| サイクル | 毎月/隔月/四半期 |
| 請求日 | 発行日 |
| 送信モード | 自動/確認/手動 |
| 期間 | 開始月〜終了月 |
| 最終発行月 | 直近で請求書を発行した月 |
| ステータス | 有効 / 停止 |

### 操作ボタン

| ボタン | 説明 |
|--------|------|
| 👁 プレビュー | 請求書のプレビュー表示 |
| 編集 | 設定を修正 |
| 有効 / 停止 | 定期請求のON/OFF切替 |
| 削除 | 定期請求を削除（管理者のみ） |

> 定期請求はCron（自動処理）により、設定された請求日に自動発行されます。承認設定が有効な場合は「承認待ち」に登録されます。

## 13. 取引先管理

取引先（お客様・仕入先）の情報を管理します。

### 取引先の登録

1. 「+ 新規作成」ボタンをクリック
2. 以下の項目を入力します

| 項目 | 説明 |
|------|------|
| 会社名 * | 取引先の名前（必須） |
| フリガナ | カタカナ表記 |
| 住所 | 所在地 |
| 電話番号 | 連絡先電話番号 |
| メール | 請求書の送信先メールアドレス |
| 請求タイプ | 即時請求 or 締日請求 |
| 締日 | 締日請求の場合、最大4つの締日を設定（末日 or 1〜28日） |
| 締日処理 | 自動送信 / 確認後送信 / 手動 |
| 事業部 | 紐づける事業部 |
| 単発フラグ | 一度きりの取引先の場合にON |

### 請求タイプの違い

| タイプ | 説明 |
|--------|------|
| **即時請求** | 納品書ごとに個別に請求書を発行 |
| **締日請求** | 設定した締日ごとに、期間内の納品書をまとめて請求書を発行 |

### CSV入出力

**CSVインポート：**
1. 「CSV取込」ボタンをクリック
2. CSVファイルを選択
3. 既存の取引先は会社名で照合して上書き更新、新規は追加

**CSV出力：**
- 「CSV出力」ボタンで全取引先をCSVダウンロード（BOM付きUTF-8）

## 14. 商品マスタ

商品・サービスの情報を管理します。納品書・請求書の明細行で使用されます。

### 商品の登録

| 項目 | 説明 |
|------|------|
| 商品名 * | 商品の名前（必須） |

## 付録: 消費税の計算方法

- 明細行ごとに税率（10%/8%/0%/カスタム）が設定されます
- 同じ税率の商品はグループ化され、税率別に消費税を計算します
- 消費税は **切り捨て**（Math.floor）で計算されます
- 合計 = 税率別小計の合計 + 税率別消費税の合計

**計算例：**
```
商品A: 1,000円 × 3個 = 3,000円（税率10%）→ 税300円
商品B: 500円 × 2個 = 1,000円（税率8%）→ 税80円
──────────────────
小計: 4,000円 / 消費税: 380円 / 合計: 4,380円
```
