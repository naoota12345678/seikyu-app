## 請求書の自動発行フロー

### cronの仕組み
- **実行時刻**: 毎日 18:00 UTC（JST 翌日 3:00）
- **1本のcronで全処理**: 締日請求・定期請求・送信予定日自動送信・楽天sync・Amazon sync

### 締日請求の流れ
1. 取引先に締日を設定（例: 14日締め）
2. 締日までに納品書を作成
3. 締日の翌日 JST 3:00 にcronが実行
4. 承認OFF → 請求書を即発行。sendMode="auto"ならメールも即送信
5. 承認ON → pendingBillingsに保存 → 承認待ちページで確認後に発行

### 定期請求の流れ
1. 定期請求を登録（取引先・品名・金額・サイクル・請求日）
2. 承認OFF → 請求日の翌日 JST 3:00 に自動発行
3. 承認ON → 請求日の10日前にpendingBillingsに保存 → 承認後に発行

### 送信予約の仕組み
1. 請求書発行時に「送信予定日」を設定
2. 請求書は作成されるがメールはまだ送らない（sentStatus="scheduled"）
3. 送信予定日の JST 3:00 にcronが実行 → メール自動送信
4. つまり「発行」と「送信」を分けられる

## 承認設定（3種類、個別ON/OFF）

| 設定 | デフォルト | 対象 |
|------|----------|------|
| 請求書発行の承認 | OFF | 手動発行（納品書→請求書、月締め） |
| 定期請求の承認 | OFF | cron自動発行の定期請求 |
| 再請求の承認 | ON | 未入金の再請求（メール/Stripe） |

- 設定画面で切り替え可能
- ONの場合 → 承認待ちページに入り、管理者が確認後に実行
- OFFの場合 → 即実行（発行 or 送信）

## 残高管理の考え方

### 基本原則
- **入金と消込は別の操作**（お金に色はない）
- 入金記録 → 残高が減る（paymentHistoryに記録）
- 消込 → 請求書のステータスがpaidになる（残高は動かない）

### 残高の計算式
**残高 = 期首残高 + 全請求書合計 − 全入金合計**

### 操作方法

#### 入金があったとき
1. 残高管理ページ → 取引先の「入金記録」ボタン
2. 入金日と金額を入力 → 保存
3. 残高が減る、paymentHistoryに記録される

#### 消込（どの請求書が入金済か記録）
1. 請求書一覧 → 状態バッジ（未収/期限超過）をクリック
2. 確認ダイアログ → 「入金済」に変更
3. もう一度クリックすれば「未収」に戻せる（トグル）

#### 入金の取消
1. 残高管理ページ → 元帳の入金行の「取消」ボタン
2. 残高が元に戻る、paymentHistoryから削除

#### 期首残高の設定
- システム導入前の未収残高を設定
- 残高管理 → 取引先の「期首残高」ボタン
- 基準日と金額を入力 → 残高に反映

#### 残高がおかしいとき
- 残高管理ページ右上の「残高再計算」ボタン
- 期首残高 + 全請求書 − 全入金 で再計算

## 取引先元帳の見方

| 列 | 意味 |
|----|------|
| 借方（赤） | 請求書発行 → 残高が増える |
| 貸方（緑） | 入金 → 残高が減る |
| 残高 | その時点での未収残高 |

- 取引先で絞り込み可能
- 期首残高は「繰越」行（青背景）で表示
- 入金行に「取消」ボタンあり
- 請求書行にステータスバッジ（入金済/期限超過/未収）

## 再請求フロー

### 未入金アラート
- 請求から1ヶ月以上経過した未収請求書がアラート表示
- 残高管理ページのバナーと、取引先行の「要確認」バッジ

### 再請求の方法
1. **請求書一覧**: 未収の請求書に「再請求▼」ドロップダウン
2. **残高管理**: 未入金アラートの取引先に「再請求▼」ドロップダウン

### 選択肢
- **メールで再請求**: 督促メール送信（PDFリンク付き、手数料なし）
- **Stripeで請求**: オンライン決済リンク付き請求書（手数料3.6%）

### 承認フロー
- 再請求の承認がON（デフォルト） → 承認待ちに追加
- 承認待ちページで「メール送信」or「Stripe送信」を選んで実行

## EC売上連携

### 楽天RMS
- 設定画面でサービスシークレット・ライセンスキーを入力
- 売上管理ページ「今日同期」/ 設定ページ「初期同期（過去1年）」
- 毎日 JST 3:00 にcronで前日分を自動同期

### Amazon SP-API
- 設定画面でClient ID・Client Secret・Refresh Tokenを入力
- 売上管理ページ「今日同期」/ 設定ページ「初期同期（過去1年）」
- 毎日 JST 3:00 にcronで前日分を自動同期

### CSV売上取込
- 売上管理ページからCSVインポート
- 形式: 日付, 金額, 件数, ソース
